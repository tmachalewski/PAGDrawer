## Key Changes

### 1. CWE REST API Refactoring (`3c54d95`)

Replaced the XML download approach with the official MITRE CWE REST API.

**Before:**
- Downloaded 30MB ZIP file from `cwe.mitre.org/data/xml/cwec_latest.xml.zip`
- Parsed XML with ElementTree
- Complex extraction of `Common_Consequences`

**After:**
- Uses REST API at `https://cwe-api.mitre.org/api/v1/`
- Simple HTTP GET requests per CWE
- JSON response with `CommonConsequences` field
- No authentication required

**API Endpoint:**
```
GET https://cwe-api.mitre.org/api/v1/cwe/weakness/{id}
```

**Response structure:**
```json
{
  "Weaknesses": [{
    "ID": "354",
    "Name": "Improper Validation of Integrity Check Value",
    "CommonConsequences": [
      {"Scope": ["Integrity"], "Impact": ["Modify Application Data"]},
      {"Scope": ["Non-Repudiation"], "Impact": ["Hide Activities"]}
    ]
  }]
}
```

### 2. Remove "Varies" Default Technical Impact (`b938008`)

When CWE enrichment is disabled or CWE data is unavailable, `technical_impact` is now `None` instead of `"Varies"`. The graph builder skips creating TI nodes for `None` impacts.

**Impact on graph:**
| Enrichment | TI Nodes | Total Nodes |
|------------|----------|-------------|
| None (before) | 128 ("Varies") | 493 |
| None (after) | 0 | 365 |
| CWE only | 92 | 463 |

### 3. Preserve Trivy Data on Config Change (`c243117`)

The `/api/config` endpoint now checks `current_data_source` and rebuilds using uploaded Trivy/deployment data instead of reverting to mock data.

**Before:** Changing singular/universal settings reloaded mock data (12 hosts)
**After:** Config changes preserve uploaded Trivy data (2 hosts from postgres scan)

### 4. Relayout Excludes Hidden Elements (`c5e9491`)

The `runLayout()` function now filters out elements with `.exploit-hidden` or `.filtered` classes before running the layout algorithm.

**Before:** "Relayout" after "Exploited Paths" kept hidden nodes in layout calculation
**After:** Layout repositions based only on visible nodes

```typescript
// Get only visible elements (exclude hidden by exploit paths or filters)
const visibleElements = cy.elements().not('.exploit-hidden').not('.filtered');
visibleElements.layout(layoutConfig).run();
```
